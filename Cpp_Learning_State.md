## 2. 学习目标

目标是系统学习 C++，补齐 C++ 工程开发能力，而不是以刷算法为主。

教学重点：

- C++ 语言机制：类型、作用域、生命周期、对象模型、模板和现代 C++ 基础。
- 工程实践：项目组织、接口设计、资源管理、错误处理和代码可维护性。
- 编译构建：命令行编译、链接、CMake、目录结构和库依赖。
- 调试测试：编译错误定位、运行时调试、断点、调用栈和单元测试。
- 小项目：用项目驱动复盘语言机制和工程能力。

教学方式要求：

- 先解释“为什么需要这个东西”，再讲“怎么写代码”。
- 每个知识点配最小可运行代码。
- 复杂概念要用具体例子逐步模拟。
- 不默认用户有扎实的 C++ 工程经验。
- 不把重点放在 LeetCode 或算法题，除非用户明确要求。

## 8. 近期需要复盘的易错点

- `std::sort` 会原地修改容器顺序，不是只返回一个排序结果。
- `std::find` 找不到时返回 `end()`，不是返回 `-1`。
- `std::string::npos` 是 `std::size_t` 的特殊值，用来表示查找失败，不应当作有效下标。
- `unordered_map` / `unordered_set` 不保证遍历顺序等于插入顺序。
- `const auto&` 的主要工程原因是只读保护和避免复制，不是为了“方便解引用”。
- 如果函数必须接收有效对象，`Player&` 通常比 `Player*` 更能表达接口约束；如果允许没有对象，才使用指针并检查 `nullptr`。
- `std::move` 本身不移动、不释放对象，只是把表达式转换成可被移动的形式；真正转移资源的是移动构造或移动赋值。
- RAII 负责资源自动释放；移动语义主要负责避免昂贵深拷贝并转移资源所有权。
- `shared_ptr` 的重点是共享所有权和延长生命周期，不是“多个变量都能操作对象”。
- 空 `optional` 强行取值属于未定义行为，不是悬空指针问题。
- Lambda 中 `std::sort` 的 `left` / `right` 只是算法拿来比较的两个元素，不应固定理解为容器中固定前后位置。
